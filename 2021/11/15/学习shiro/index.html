学习shiro

发表于 2021-11-15 更新于 2021-11-25 分类于 web安全

如题

本来看了一些博客，还记了一些笔记，兜兜转转发现不如读一遍原版文档来的清晰明了。

https://shiro.apache.org/reference.html

然后跟着大佬博文走一遍就行

  • CVE-2020-11989
  • Apache Shiro < 1.5.3

?：匹配一个字符
*：匹配零个或多个字符串
**：匹配路径中的零个或多个路径

org.apache.shiro.web.util.WebUtils#getRequestUri 会对url进行2次解码
然后会分配一个dologin的matcher
然后会比较pathPattern，dologin的pattern为/admin/*

比较的点在 org.apache.shiro.util.AntPathMatcher#doMatch
因为 xxx%252fxxx 二次解码之后是 xxx/xxx 与 * 无法匹配 (*只能匹配一层路径)
然后就返回FALSE，表示这个路径不归我管，就绕过了

我们的控制器路径是 /admin/{name}
最终 进入 /admin/xxx%2fxxx

  • CVE-2020-13933
  • Apache Shiro < 1.6.0
    对11989的反击

/org/apache/shiro/web/util/WebUtils.java:244

会对;后面的进行截断，导致URL变成 /admin/ 自然匹配不到 /admin/*

  • CVE-2020-17523
  • Apache Shiro < 1.7.1
    对13933的反击

org.apache.shiro.util.StringUtils#tokenizeToStringArray(java.lang.String, java.lang.String, boolean, boolean)
第三个参数控制是否进行trim()操作，代码里硬编码为true，于是空格就没了，自然匹配不到 /admin/*
