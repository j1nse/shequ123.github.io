中间件漏洞总结

发表于 2019-04-09 | 更新于 2019-04-10 | 分类于 web安全

IIS

IIS5.x/6.0

1.目录解析
建立xx.asp为名称的文件夹，将asp文件放入，访问/xx.asp/xx.jpg，其中xx.jpg可以为任意文件后缀，即可解析

2.文件解析
后缀解析：/xx.asp;.jpg /xx.asp:.jpg(此处需抓包修改文件名)

3.默认解析：
/xx.asa，/xx.cer，/xx.cdx

4.PUT漏洞
开启WebDAV拓展后，可以用PUT方法上传文件

IIS 7.0/IIS 7.5/ Nginx <8.03畸形解析漏洞

在默认Fast-CGI开启状况下,上传一个名字为wooyun.jpg，内容为
<?PHP fputs(fopen('shell.php','w'),'<?php eval($_POST[cmd])?>');?>
的文件，然后访问wooyun.jpg/.php,在这个目录下就会生成一句话木马 shell.php

Nginx <8.03 空字节代码执行漏洞

在一个文件路径(/xx.jpg)后面加上%00.php会将 /xx.jpg%00.php 解析为 php 文件。

Apache

Apache将从右至左开始判断后缀，若x3非可识别后缀，再判断x2，直到找到可识别后缀为止，然后将该可识别后缀进解析
test.php.x1.x2.x3则会被解析为php
php|php3|phtml|php4|php5

常用命令

一般

shutdown -h now 关机
tar -xvf 解压
tar -zvf 压缩
whereis find 位置 -name 文件搜索 | whereis从数据库查找 find默认在当前目录下寻找
which 命令搜索
ls -la l详细 a全部
ctrl l 清理屏幕
ctrl u 清理命令
ctrl z 放进后台
ctrl r 搜索历史命令
history -c 清空历史命令
!!重复执行上一条命令
!n重复执行第n个命令
!xxxx 重复执行以xxxx开头的命令
cat -n 附带行号
tac 倒着输出
tail -n 看最后n行 -c为字节 -f持续查看追加内容
od以二进制形式读取
more less 翻页查看
sed

常用

lastlog 所有用户最后一次登陆时间
w 显示当前登陆用户和正在执行的指令 pts/0~n表示远程登陆，win最多支持2~4,linux支持很多 tty1~n表示本机登陆
useradd xx 添加用户xx
groupadd xx 添加新工作组xx

shell

权限

w=4;r=2;x=1
目录最大权限666 文件最大权限777
chmod 改权限
chown 改文件用户组
lsattr 查看文件隐藏属性
chattr 改变文件隐藏属性
umask 改文件默认权限
suid chmod 4775 chmod u+s
sgid 既可以针对目录，也可以针对文件，用户必须对此目录有r和x权限，才能进入目录;如果是文件，身份会变成所属组;如果是目录，任何用户在这个目录创建的文件所有者会变成当前目录所有者 chmod 2775
sbid 只针对目录有效，普通用户对此目录有写入权限;设置之后，普通用户只能添加文件和删除自己的文件
