linux运维安全相关

发表于 2019-04-07 | 更新于 2019-04-12 | 分类于 web安全

常用命令

一般

shutdown -h now 关机
tar -xvf 解压
tar -zvf 压缩
whereis find 位置 -name 文件搜索 | whereis从数据库查找 find默认在当前目录下寻找
which 命令搜索
ls -la l详细 a全部
ctrl l 清理屏幕
ctrl u 清理命令
ctrl z 放进后台
ctrl r 搜索历史命令
history -c 清空历史命令
!!重复执行上一条命令
!n重复执行第n个命令
!xxxx 重复执行以xxxx开头的命令
cat -n 附带行号
tac 倒着输出
tail -n 看最后n行 -c为字节 -f持续查看追加内容
od以二进制形式读取
more less 翻页查看
sed

常用

lastlog 所有用户最后一次登陆时间
w 显示当前登陆用户和正在执行的指令 pts/0~n表示远程登陆，win最多支持2~4,linux支持很多 tty1~n表示本机登陆
useradd xx 添加用户xx
groupadd xx 添加新工作组xx

shell

权限

w=4;r=2;x=1
目录最大权限666 文件最大权限777
chmod 改权限
chown 改文件用户组
lsattr 查看文件隐藏属性
chattr 改变文件隐藏属性
umask 改文件默认权限
suid chmod 4775 chmod u+s
sgid 既可以针对目录，也可以针对文件，用户必须对此目录有r和x权限，才能进入目录;如果是文件，身份会变成所属组;如果是目录，任何用户在这个目录创建的文件所有者会变成当前目录所有者 chmod 2775
sbid 只针对目录有效，普通用户对此目录有写入权限;设置之后，普通用户只能添加文件和删除自己的文件

攻击

入侵 Linux 服务器后需要清除哪些日志
web日志，如apache的access.log,error.log。 直接将日志清除过于明显,一般使用sed进行定向清除

e.g. sed -i -e ‘/192.169.1.1/d’

history命令的清除，也是对~/.bash_history进行定向清除

wtmp日志的清除，/var/log/wtmp

登录日志清除 /var/log/secure
